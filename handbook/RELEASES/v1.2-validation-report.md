## 1. Release objective

Version 1.2.0 establishes one controlled company identity and two deterministic
licensing profiles. Current and future R packages use GPL version 2 or later
with an explicit dependency copyright boundary. Websites, WAEF, the company
policy repository, and the repository template use the proprietary profile.

## 2. Repository coverage

The validated rollout covers the company policy repository, WAEF,
repository-template, DCC, IRTC, WFC, mergecalib, ratecalib, website, and
website-global-preview.

The five R packages contain:

- parseable DESCRIPTION metadata with `License: GPL (>= 2)`;
- the controlled maintainer and company contact identities;
- the GPL version 2 repository text;
- `inst/COPYRIGHTS` with separately distributed dependency boundaries; and
- the controlled CODEOWNER.

The five proprietary repositories contain the controlled company copyright
holder, 2026 notice, licensing contact, and CODEOWNER.

## 4. Consistency audit

The active source scan excluded Git internals, generated output, vendored
assets, built archives, and the historical ratecalib 0.3.0 archive. It found no
personal Gmail address, example maintainer address, obsolete English legal
name, or known Chinese legal-name typo.

Profile-presence checks passed for all ten repositories. Whitespace validation
passed, and no build directory, CodeGraph data, secret, credential, client
data, or unrelated generated file was included in the committed changes.

The unpublished ratecalib 0.3.1 source archive was rebuilt from the standardized
package source. The historical 0.3.0 archive was not modified.

## 5. Environmental limitations

- The local environment did not provide markdownlint-cli2 or lychee, so the
  repository-template CI variants of those checks were not run locally.
  Deterministic profile-asset and forbidden-placeholder checks passed.
- External links were not tested over the network.
- No GitHub Actions run, remote push, CRAN submission, deployment, or Git
  history rewrite was performed.
- This report records engineering consistency checks and does not replace
  jurisdiction-specific legal review.

## 6. Publication decision

The v1.2.0 policy, profiles, templates, package metadata, dependency boundaries,
and proprietary notices satisfy the approved local rollout criteria. The
document set is suitable as the active company copyright and licensing
baseline, subject to the environmental limitations above.
